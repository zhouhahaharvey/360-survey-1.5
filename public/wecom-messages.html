技术规范

企业微信模板卡片 API 的完整字段说明

文本通知型 text_notice

  • card_type: "text_notice"
  • main_title.title: 主标题，最多128字节
  • main_title.desc: 副标题，最多512字节
  • emphasis_content: 关键数据，只能有1个
  • sub_title_text: 二级内容，最多512字节
  • jump_list: 跳转链接数组

通用限制

  • 所有 URL 必须是 HTTPS
  • 不支持 HTML 标签
  • 不支持自定义样式（颜色、字体）
  • 消息跳转只能到小程序或 URL
  • source_desc 最多30字符
  • 单个消息大小不超过 2048 字节

使用建议

基于实际项目经验的最佳实践

关键数据突出显示

使用 emphasis_content 突出最重要的信息，如截止时间或待办数量。注意：只能有一个关键数据。

按钮文字精简

按钮文字控制在16字节以内（约6-8个汉字），避免过长被截断。

避免超出发送限制

发送前检查文本长度，超长的内容会导致消息发送失败。中文字符按3字节计算。

必须使用HTTPS链接

所有跳转链接必须是HTTPS协议，HTTP链接会被拒绝。确保域名已备案。

不支持富文本格式

不要在内容中使用HTML标签（如<b>、<br>），企业微信会直接显示为纯文本。

常见错误

开发中容易遇到的问题及解决方案

错误：包含多个 emphasis_content

企业微信 text_notice 模板只支持一个关键数据。如需展示多个信息，请使用 sub_title_text 字段。

错误：URL 使用 HTTP 协议

企业微信要求所有跳转链接必须是 HTTPS。请确保使用有效的 SSL 证书，且域名已完成备案。

错误：标题或描述超出长度限制

标题最多128字节，描述最多512字节（1个中文约3字节）。请使用工具预先计算字节长度。

错误：使用未授权的卡片类型

部分卡片类型需要申请权限。请确保在企微管理后台已启用对应的模板卡片能力。
