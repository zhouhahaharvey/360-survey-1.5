## 📱 推荐体验流程

### 完整业务流程体验

#### 1. HRBP发起调研
1. **仪表盘** - 查看数据统计和快捷入口
2. **发起调研** - 选择场景，体验 Step1→Step2→Step3 完整流程
3. **提交审核** - Step3提交后进入审核状态
4. **我的待审核** - 查看已提交审核的调研

#### 2. 管理员审核
1. **审核中心** - 查看所有待审核调研
2. **批量审核** - 勾选多个调研进行批量审核
3. **选择推送方式**:
   - 立即推送：直接发起调研
   - 加入待推送池：等待合并推送
4. **待推送池** - 合并多个调研统一推送

#### 3. 员工填写评估
1. **企微入口** - 访问 `respondent/todo-list.html?from=wechat`
2. **待办列表** - 查看场景化卡片
3. **问卷填写** - 分维度评分，填写主观评价
4. **连续填写** - 提交后自动进入下一份

#### 4. 查看报告
1. **报告中心** - 查看已完成调研的报告
2. **调研详情** - 查看进度统计和操作按钮
3. **报告编辑器** - 在线编辑和导出

## ✨ 核心功能详解

### 1. 审核流程（v1.4新增）

```
HRBP发起 → 提交审核 → 管理员审核 → 发起调研
```

- **发起人**: 创建调研后提交审核，可在"我的待审核"查看状态
- **管理员**: 在"审核中心"批量审核，可立即推送或加入待推送池
- **状态流转**: draft → pending_review → relation_approved → active → completed

### 2. 待推送池（v1.4新增）

将多个调研合并后统一推送，减少对同一评估人的重复消息打扰。

**适用场景**:
- 同一部门多个人同时晋升评估
- 批量转正评估统一推送
- 减少企微消息轰炸

**操作步骤**:
1. 审核时选择"加入待推送池"
2. 进入待推送池页面
3. 勾选需要合并的调研
4. 一键合并推送

### 3. 双视角催办（v1.4新增）

支持两种催办视角:

| 视角 | 说明 | 适用场景 |
|------|------|---------|
| 按被考评人 | 查看每个被评估人的整体进度 | 了解某人评估完成情况 |
| 按考评人 | 查看每个评估人的待办列表 | 定向催办特定评估人 |

**智能合并**: 同一评估人多条待办合并为一条消息

### 4. 连续填写（v1.4新增）

员工填写评估时，提交后自动跳转到下一份待办，提升填写效率。

**实现逻辑**:
1. 检测是否还有未完成的评估任务
2. 有则显示"提交并进入下一份"按钮
3. 提交后自动跳转到下一个待办

## 📝 使用提示

1. **首次访问**: 建议先清除浏览器缓存，确保加载最新数据
2. **填写端**: 页面请在移动端浏览器或开启开发者工具的移动设备模式下查看
3. **审核流程**: 从"发起调研"开始，完整体验 Step1→Step2→Step3→审核→推送
